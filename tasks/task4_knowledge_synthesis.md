# Task 4: Knowledge Synthesis Engine (Kimi)

## 1. Description
This is a **Fat Node** (Reasoning). It acts as the final aggregator. It takes a list of paths to the intermediate Markdown files generated by Task 3 and uses the **Kimi API (via Anthropic compatibility layer)** to synthesize them into a single, cohesive Markdown document.

## 2. DSPy / Schema-Driven Contract

### Signature (Input)
```json
{
  "type": "object",
  "properties": {
    "intermediate_md_files": {
      "type": "array",
      "items": { "type": "string", "description": "Paths to the intermediate .md files" }
    }
  },
  "required": ["intermediate_md_files"]
}
```

### Signature (Output Schema)
Because the Anthropic protocol might not natively support OpenAI's strict `response_format={"type": "json_object"}`, we will rely on structured prompt instructions or use Kimi's native tool calling if strict JSON is needed. However, since the goal is pure Markdown, we can simply extract the text content directly.

```json
{
  "type": "object",
  "properties": {
    "final_markdown": { "type": "string" }
  },
  "required": ["final_markdown"]
}
```

## 3. Execution Details
*   Read all `.md` files specified in `intermediate_md_files`.
*   Concatenate them (optionally with headers indicating the source/page).
*   Initialize the `anthropic` Python client with `base_url="https://api.kimi.com/coding/"`.
*   Pass the concatenated string to Kimi with instructions to organize, synthesize, format the final document (keeping Mermaid diagrams intact), and absolutely avoid outputting any conversational filler.
